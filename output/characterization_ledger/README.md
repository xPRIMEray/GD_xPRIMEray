# Characterization Ledger

Short name: **Fixture Runs — Characterization Ledger**

## Purpose

Machine-readable ledger tracking characterization status across all fixture runs. Records which fixtures have been traced, committed, have SSIM scores, and which are missing or have import errors. Used to audit completeness of the fixture test suite.

## Source / Generation Context

- Auto-generated by the fixture characterization harness
- Imports data from `fixture_runs/` outputs

## What the Output Shows

A set of `.translation` files tracking per-fixture status:
- `fixture_runs.status.translation` — Pass/fail per fixture
- `fixture_runs.ssim.translation` — SSIM score per fixture (visual similarity to reference)
- `fixture_runs.traced.translation` — Which fixtures have been traced
- `fixture_runs.commit.translation` — Which fixtures have committed output
- `fixture_runs.miss.translation` — Missing fixtures
- `fixture_runs.total.translation` — Total counts

## Key Files

- `fixture_runs.status.translation` — Status summary
- `fixture_runs.ssim.translation` — SSIM scores
- `fixture_runs.miss.translation` — Missing fixtures
- `fixture_runs.csv.import` — CSV import record
